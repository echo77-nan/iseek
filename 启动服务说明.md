# 启动服务说明

## 错误信息
```
ModuleNotFoundError: No module named 'app'
```

## 原因
在错误的目录下运行了 uvicorn 命令。需要在 `backend` 目录下运行。

## 正确的启动方式

### 方法1：在 backend 目录下启动（推荐）

```bash
cd /home/echo.ln/iseek/backend
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### 方法2：从项目根目录启动

```bash
cd /home/echo.ln/iseek
uvicorn backend.app.main:app --reload --host 0.0.0.0 --port 8000
```

### 方法3：使用 Python 模块方式启动

```bash
cd /home/echo.ln/iseek
python3 -m uvicorn backend.app.main:app --reload --host 0.0.0.0 --port 8000
```

## 验证当前目录

启动前确认当前目录：

```bash
pwd
# 应该显示：/home/echo.ln/iseek/backend

# 或者检查文件是否存在
ls -la app/main.py
# 应该能看到文件
```

## 完整的启动流程

```bash
# 1. 进入 backend 目录
cd /home/echo.ln/iseek/backend

# 2. 确认文件存在
ls -la app/main.py

# 3. 启动服务
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

## 如果使用虚拟环境

```bash
cd /home/echo.ln/iseek/backend
source venv/bin/activate  # 如果创建了虚拟环境
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```
