# 修复前端启动脚本

## 问题
服务器上的 `package.json` 缺少 `start` 脚本，导致 `npm start` 失败。

## 解决方案

### 方法1：在服务器上直接修改 package.json

在服务器上执行：

```bash
cd /home/echo.ln/iseek/frontend

# 编辑 package.json，在 scripts 部分添加 "start": "vite"
nano package.json
# 或
vi package.json
```

在 `scripts` 部分添加：
```json
"start": "vite",
```

完整的 scripts 部分应该是：
```json
"scripts": {
  "dev": "vite",
  "start": "vite",
  "build": "vite build",
  "preview": "vite preview"
}
```

### 方法2：使用 sed 命令快速添加

```bash
cd /home/echo.ln/iseek/frontend

# 在 "dev" 行后添加 "start" 脚本
sed -i '/"dev": "vite",/a\    "start": "vite",' package.json
```

### 方法3：使用 cat 命令重新创建 scripts 部分

```bash
cd /home/echo.ln/iseek/frontend

# 备份原文件
cp package.json package.json.bak

# 使用 sed 替换 scripts 部分
sed -i 's/"dev": "vite",/"dev": "vite",\n    "start": "vite",/' package.json
```

### 方法4：从本地同步文件

如果本地文件已更新，可以使用 scp 同步：

```bash
# 在本地执行
scp frontend/package.json user@server:/home/echo.ln/iseek/frontend/package.json
```

## 验证修复

修复后验证：

```bash
cd /home/echo.ln/iseek/frontend

# 检查 scripts
cat package.json | grep -A 5 "scripts"

# 应该看到 "start": "vite"
```

## 启动服务

修复后启动：

```bash
cd /home/echo.ln/iseek/frontend
npm start
```

或者使用：

```bash
npm run dev
```
